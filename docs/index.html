Vibe-Core v1.5.1

A headless Linux audio engine using Shared Memory IPC. All nodes reside in /dev/shm/vibe/ for zero-copy efficiency.

1. The Engine

Optimized for legacy hardware (AMD E-350 / 1.6GHz). Vibe-Core prioritizes thermal headroom over high-frequency UI updates.

  • Dynamic Batching: Decodes 10 chunks (~200ms) per cycle to reduce context switching.
  • Adaptive Sleep: Logic adjusts based on len(audioBuffer) (Target: 512 chunks).
  • Telemetry Sync: 500ms sync window (Increases efficiency on older CPUs).

2. Control Interface (FIFOs)

Write-only nodes that trigger immediate engine actions via mkfifo.

Command Pipe (ctl)

echo "pause" > ctl # Suspend audio thread
        echo "resume" > ctl # Wake audio thread
        echo "next" > ctl # Advance playlist index
        echo "prev" > ctl # Decrement playlist index
        echo "shuffle" > ctl # Fisher-Yates shuffle toggle
        echo "exit" > ctl # Graceful cleanup of /dev/shm nodes

Extended Controls

echo "45.5" > seek # Jump to timestamp
        echo "/path/to/music.mp3" > play_now # Priority interrupt play

3. Stateful & Telemetry Nodes (Shared Memory)

Node Type Format Description
vol R/W 0-200 Linear gain. 100-200: Boost Mode. Initialized at 50.
state RO string playing | paused
now_playing RO string Filename of the active track.
head RO float Elapsed time (seconds). Updated @ 500ms.
len RO float Total duration of the active file.
db RO float RMS Intensity (-100 to 0). Used for meters.
shuffle_state RO string on | off
playlist RO string Newline-separated queue of paths.

4. Low-Level Implementation

Entropy-Based Memory Triage

Internal sampleSum calculations occur in the onSamples callback. By calculating RMS values during the decode pass, Vibe-Core avoids a second pass over the data, saving valuable CPU cycles on Bobcat-era cores.
