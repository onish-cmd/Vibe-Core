Vibe-Core v1.4.3

A headless Linux audio engine using Memory-Mapped IPC. All nodes reside in /dev/shm/vibe/.

1. Control Interface (FIFOs)

These nodes use mkfifo. Writing to them triggers immediate engine reactions.

Command Pipe (ctl)

echo "pause" > ctl # Suspend playback
        echo "resume" > ctl # Resume playback
        echo "next" > ctl # Skip to next track
        echo "shuffle" > ctl # Toggle Fisher-Yates (Writes to shuffle_state)
        echo "exit" > ctl # Cleanup and terminate engine
        echo "prev" > ctl # Go to previous track

Seek Control (seek)

Instantly drains the audio buffer and repositions the MP3 decoder.

echo "45.5" > seek # Jump to 45.5 seconds

Priority Injection (play_now)

Interrupts current playlist, saves the index, and plays a specific file. Returns to original state after.

echo "/home/onish/music/alert.mp3" > play_now

2. Stateful & Telemetry Nodes

Node Type Format Description
vol R/W 0-100 Linear gain. Initialized at 50.
state RO string playing | paused
now_playing RO string Filename of active MP3.
head RO float Current elapsed time (seconds).
len RO float Total track duration (seconds).
db RO float RMS Logarithmic intensity (-100 to 0).
shuffle_state RO string on | off
playlist RO string (each song in playlist is separated by newline) Current playlist

3. Internal Logic

Automatic Refresh (fsnotify)

The engine uses inotify to watch the music/ directory. Adding or removing files sets needsRefresh = true, triggering a playlist rebuild at the next track transition.

PCM Energy Calculation

Calculated in the onSamples callback via sampleSum += res * res. Provides zero-latency data for visualizers without extra CPU passes. It is not a spectrum due to high CPU usage with spectrum, this is optimized for older CPUs like the AMD E-350 this was made on.
