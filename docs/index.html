Vibe-Core v1.3.0

An ultra-low latency, headless audio engine for Linux. Designed for FOSSHack 2026, Vibe-Core treats the filesystem as the interface, enabling seamless IPC across any language.

Headless Architecture

Vibe-Core operates as a pure daemon. Interaction occurs through Memory-Mapped Control Nodes in /dev/shm/vibe. By leveraging tmpfs, control signals and telemetry are exchanged at RAM speeds, bypassing disk I/O bottlenecks—critical for low-spec hardware like the AMD E-350.

Control Endpoints (Write-Only)

1. Command Pipe (ctl)

A FIFO Named Pipe for real-time engine instructions. 

echo "next" > ctl    # Skip current track
echo "prev" > ctl    # Previous track (Index - 2 logic)
echo "pause" > ctl   # Suspend audio device
echo "resume" > ctl  # Resume audio device
echo "exit" > ctl    # Trigger engine cleanup and shutdown

2. Seek Control (seek)

A FIFO Named Pipe for high-precision PCM stream positioning. The engine flushes the internal audio buffer instantly upon valid seek. 

echo "30" > seek      # Jump to 30.0s
echo "0" > seek       # Restart current song

3. Priority Injection (play_now)

Interrupt Handler: Passing an absolute path immediately halts the current sequence to play the target file. The engine restores the previous playlist state and index upon completion. 

echo "/path/to/urgent_vibe.mp3" > play_now

4. Energy Telemetry (db)

Real-time Root Mean Square (RMS) Calculation: Calculated in the 20Hz telemetry loop to ensure zero jitter in the audio thread. Measures relative signal intensity against the 16-bit digital ceiling.

  • 0.0 dB: Full-scale digital peak (Hard Clipping limit).
  • -12.0 to -6.0 dB: Dynamic range "Sweet Spot".
  • -100.0 dB: Digital floor/Silence.
cat db  # Output: -18.4

Stateful Nodes (Read/Write)

5. Volume Node (vol)

Linear gain control. Writing to this node updates the internal mixer multiplier.

  • Range: 0 - 100 (Standard) | 101 - 200 (Digital Boost) 
echo "85" > vol      # Set volume to 85%

Telemetry Nodes (Read-Only)

Node Description Format
state Engine playback status playing | paused
now_playing Filename of active stream String
head Current playback position Float (Seconds)
len Total duration of stream Float (Seconds)
db Logarithmic energy level (dBFS) Float (dB) | Range: -100 to 0

Hardware Benchmarks (E-350 @ 1.6GHz)

  • CPU Overhead: ~19.5% (MP3 Decoding + Telemetry Goroutine)
  • Memory: < 15MB Resident Set Size
  • IPC Latency: Near-zero (RAM-backed Nodes)
